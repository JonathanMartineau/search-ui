Class Recommendation

This component is a SearchInterface that will display recommendations based on the user history. To get recommendations, the page view script must also be included in the page. View: https://github.com/coveo/coveo.analytics.js This component listens when the main search interface generates a query and it generates another to get the recommendations at the same time.

This component can be included in another SearchInterface, but you need to initialize the recommendation component with Coveo('initRecommendation'), before the parent SearchInterface.

linkSearchUid

linkSearchUid: boolean
  • Defined in src/ui/Recommendation/Recommendation.ts :67

Specifies if the results of the recommendation query should have the same searchUid as the ones from the main search interface query. It is used to give info to the Analytics The default value is true

mainSearchInterface

mainSearchInterface: HTMLElement
  • Defined in src/ui/Recommendation/Recommendation.ts :46

optionsToUse

optionsToUse: Array<string>
  • Defined in src/ui/Recommendation/Recommendation.ts :73

Specifies which options from the main QueryBuilder to use in the triggered query. The default value is ["expression", "advancedExpression", "constantExpression", "disjunctionExpression"]

userContext

userContext: any
  • Defined in src/ui/Recommendation/Recommendation.ts :53

Specifies the user context to send to Coveo analytics. It will be sent with the query alongside the user history to get the recommendations. If the option is not present, this component will display the same results as the main search interface.
